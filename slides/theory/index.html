Technology?

It's ready, seriously.

Technology is ready. For the past few years there were issues with many things like performance, working offline, audio support or monetisation - all that is history. Today the only thing that holds you back from building great HTML5 games is your imagination.

Web API

  • Fullscreen
  • Pointer Lock
  • Gamepad
The are APIs that can make the web game look and work as a native one - run fullscreen without the address bar, hide the cursor with the Pointer Lock API, allow to play the game with the gamepad controller.

Mobile API

  • Vibration
  • Screen orientation
  • Battery status
  • WebSMS
  • Proximity sensor
There are also mobile APIs that do the same - help harness the power of hardware from within your game like the Vibration API, using Screen Orientation, or even being able to check the battery level or send SMS.

List of APIs

WebTelephony, Vibration API, WebSMS, Idle API, Screen Orientation, Settings API, Power Management API, Mobile Connection API, TCP Socket API, Geolocation API, WiFi Information API, Device Storage API, Contacts API, Mouse Lock API, Open WebApps, WebBluetooth, Network Information API, Battery Status API, Alarm API, Browser API, Time⁄Clock API, Web Activities, Push Notifications API, Permissions API, WebFM API, FileHandle API, Network Stats API, WebPayment, IndexedDB, Archive API, Ambient light sensor, Proximity sensor, SystemXHR, ...

The list of APIs is huge, but don't be intimidated by it - API can be very easy to use.

Vibration API

window.navigator.vibrate(200);
Vibration is the API that can be implemented in just a single line of code. By writing that line you make the device vibrate for the given amount of time (in miliseconds).

Screen orientation

        checkOrientation = function() {
          if(window.orientation == 0 || window.orientation == 180) {
            // "rotate your device"
          }
          else {
            // "keep playing"
          }
        }
Screen Orientation is another interesting, easy to use API. If you have a game that can be played only in portrait or landscape modes you can check whether player holds the device in the correct orientation - if he isn't, you can show him the message to do so.

Page visibility

        document.addEventListener("visibilitychange", function() {
          if(document.hidden) {
            // "app is not visible"
          }
          else {
            // "app is visible"
          }
        });
Page Visibility API can be useful to check if the player is currently playing the game. If he moves to another tab in the browser and yours is not visible to him, you can do something in that case like for example turn off the sounds and turn them on when he returns.

Tools

Free, easy to use

There are various tools to help you with game development. Most of them are free and easy to use. If you're a front-end developer you already know the tools and workflows and don't have to learn anything new.

Writing JavaScript code

Notepad, Sublime Text, Eclipse

To write JavaScript code of your game you need an editor. It can be as plain and simple as a notepad. There are more advanced editors like Sublime Text and even IDEs like Eclipse. It really depends on your personal preferences.

Testing the game

Browser + dev tools

You can test the game using the available tools - inside the browser, with Firebug or native development tools built into the browser.

Phaser

phaser.io - free, open sourced on GitHub

The most popular framework for HTML5 game developers right now is Phaser. There's a huge community helping with the framework itself, writing articles and tutorials, building tools and editors. It is free, open sourced and is available on GitHub.

Frameworks

html5gameengine.com

There are many other interesting frameworks. You can check the website above - feel free to experiment with them and pick whatever suits you best.

Compare

jsbreakouts.org

You can also compare implementations of one given game in various frameworks to see differences between them and pick the one that have the best structure for you and your game.

Firefox OS

  • Mobile OS written in JavaScript
  • Created and maintained by Mozilla
  • Open source, available on GitHub
  • Hardware platform for the mobile web
Firefox OS is the mobile operating system written entirely in JavaScript. It's the perfect hardware platform for HTML5 games. You have access to the APIs and functions of the phone directly from JavaScript.

  • Remote debugging
WebIDE is the tool built into the browser helping you manage games, edit their source code, test them in the Simulator and even debug remotely on an actual device.

Prepare your game
for Firefox Marketplace

Manifest file and submit form

It's very easy to publish your game in the Firefox Marketplace - all you need is the manifest file and the submit form.

Manifest file

manifest.webapp 
        {
          "name": "Name of the app",
          "description": "Description of the app",
          "launch_path": "/",
          "icons": {
            "128": "/img/icon-128.png"
          },
          "developer": {
            "name": "Enclave Games",
            "url": "http://enclavegames.com"
          },
          "default_locale": "en"
        }
The manifest file is a simple JSON file containing the basic information about your game. The code above is the minimal working example.

Submit form

marketplace.firefox.com/developers/submit/

Using the submit form is easy, validating your game takes only a few days and if something goes wrong you'll get the actual feedback on how to fix it.

Install from the Web

        var installapp = navigator.mozApps.install(manifestURL);
        installapp.onsuccess = function(data) {
          // "App was installed"
        };
        installapp.onerror = function() {
          // "App was NOT installed,
          // check the installapp.error.name for more info"
        }
You don't even have to publish your game in the Marketplace to have people play it on the Firefox OS devices. You don't have to point them there to install the game - you can do that directly from your website.
